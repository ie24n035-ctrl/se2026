# 画面３つの説明
1.ログイン画面
・ユーザーがIDとパスワードを入力する画面
・認証成功時、ダッシュボード画面へ遷移する
・認証失敗時、エラーメッセージを表示する
2.ダッシュボード画面
・登録されているデータの一覧を表示する
・各データの概要（例：名前、合計点）を表示
・「新規作成」「詳細表示」ボタンを配置
3.詳細・編集画面
・選択したデータの詳細情報を表示する
・データの編集・削除が可能
・更新後はダッシュボードへ戻る
# データ
1.ユーザーデータ
・ユーザーID
・名前
・パスワード
2.成績データ
・データID
・科目
・点数
3.合計データ
・ユーザーID
・合計点
# 処理の流れ
1.ユーザーがIDとパスワードを入力
2.入力情報をサーバに送信
3.データベースのユーザー情報と照合
4.一致すればログイン成功
5.ダッシュボード画面へ遷移
6.不一致ならエラーメッセージ表示
# システム構成
フロントエンド：ユーザーインターフェース（HTML/CSS/JavaScript）
バックエンド：処理ロジック（Python / Node.jsなど）
データベース：データ保存（MySQL / SQLiteなど）
