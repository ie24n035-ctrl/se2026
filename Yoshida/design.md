1. 画面設計
① ログイン画面
ユーザーがIDとパスワードを入力する画面
認証成功時、ダッシュボード画面へ遷移する
認証失敗時、エラーメッセージを表示する
② ダッシュボード画面
登録されているデータの一覧を表示する
各データの概要（例：名前、合計点）を表示
「新規作成」「詳細表示」ボタンを配置
③ 詳細・編集画面
選択したデータの詳細情報を表示する
データの編集・削除が可能
更新後はダッシュボードへ戻る
2. データ設計
① ユーザーデータ（User）
項目名	型	説明
id	int	ユーザーID
name	string	名前
password	string	パスワード
② 成績データ（Score）
項目名	型	説明
id	int	データID
user_id	int	ユーザーID
subject	string	科目
score	int	点数
③ 合計データ（Total）
項目名	型	説明
user_id	int	ユーザーID
total_score	int	合計点
3. 処理の流れ（ログイン処理）
ユーザーがIDとパスワードを入力
入力情報をサーバに送信
データベースのユーザー情報と照合
一致すればログイン成功
ダッシュボード画面へ遷移
不一致ならエラーメッセージ表示
4. システム構成
構成概要
フロントエンド：ユーザーインターフェース（HTML/CSS/JavaScript）
バックエンド：処理ロジック（Python / Node.jsなど）
データベース：データ保存（MySQL / SQLiteなど）
